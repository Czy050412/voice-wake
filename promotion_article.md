# 我花了36小时，给电脑装了个"狗蛋"——一个完全离线的开源语音助手

> 喊一声"狗蛋"，它就能帮你查天气、开软件、控制音量，甚至自动把你想说的话发给AI对话窗口。全程离线，不要VPN，不调任何付费API。

---

## 起因：我想要一个真正"听话"的电脑

作为一个独立开发者，我日常在UE5蓝图里摸爬滚打，两个显示器开着十几个窗口，手在键盘鼠标之间飞来飞去。有时候我就想：**能不能直接喊一嗓子，让电脑自己干活？**

市面上的语音助手我基本都试过：
- **小娜/Cortana**：已入土
- **各大输入法语音**：要联网，反应慢，准确率拉胯
- **讯飞/百度**：要API Key，要钱，而且我不想把每句语音都发到云端
- **Siri/Alexa**：这是Windows，别闹

结论：**没有一个能打的。** 于是决定自己造。

---

## 翻车日记：我换了四个识别引擎，炸了无数次

### 第一站：Google Web Speech API
Python 装个 `SpeechRecognition`，调 Google 免费 API，中文识别还行。**但问题是——国内没VPN根本用不了。** 而且网络延迟感人，喊完"狗蛋"等两秒才反应。

### 第二站：Vosk 离线模型
"既然Google要VPN，那我用离线的。" 下了 Vosk 中文小模型，42MB，不用联网，爽。**然后发现中文识别准确率约等于随机数生成器。** 我说"打开浏览器"，它识别成"打击力气"，我说"查天气"，它给我"炸田鸡"。

### 第三站：FunASR（阿里达摩院 Paraformer）
这个是真正的硬货——6万小时普通话训练，准确率95%+。但模型944MB，托管在国内 ModelScope。**下载速度让我梦回拨号上网时代**，200KB/s反复超时，下了三次全断在200MB左右。

### 第四站：faster-whisper（最终方案）
OpenAI Whisper 的 CTranslate2 加速版。small 模型 500MB，HuggingFace 下载13秒搞定（开了VPN）。**CPU 上 int8 量化跑，速度飞快，中文准确率90%+。** 而且模型下载一次就缓存本地，之后完全离线。

**四换引擎，数次翻车，终于稳了。**

## 它能干什么？

### 基础操作：喊就完事

```
"狗蛋 帮我查徐州天气"
"狗蛋 打开QQ音乐"
"狗蛋 音量调大"
"狗蛋 锁屏"
"狗蛋 截图"
"狗蛋 现在几点"
```

**系统命令和AI对话自动分流。** 说"打开微信"→本地执行，说"帮我写个Python脚本"→自动粘贴到CodeBuddy/WorkBuddy对话框并发送。

### 整条链路长这样

```
麦克风 → SpeechRecognition VAD → faster-whisper 识别
         ↓
   检测唤醒词"狗蛋"
         ↓
   系统命令? → 本地执行（开软件/调音量/锁屏...）
   AI对话?  → pyautogui 点击输入框 → 粘贴 → 回车发送
```

### 配合 CodeBuddy/WorkBuddy，战斗力翻倍

我用的是 WorkBuddy（腾讯云AI编程助手），关掉窗口、干别的事，喊一声"狗蛋"它自己弹回来、自己粘贴、自己发送。**真正的语音召唤。**

## 技术栈

| 层 | 技术 | 为什么选它 |
|------|------|-----------|
| 语音识别 | faster-whisper (small) | CTranslate2加速，CPU可用，准确率高 |
| 音频采集 | PyAudio + SpeechRecognition | 自带VAD语音活动检测 |
| 窗口操控 | Win32 API + pyautogui | 精确定位窗口，模拟鼠标键盘 |
| 配置系统 | YAML | 可视化参数，不用改代码 |
| 系统命令 | 自定义插件模块 | 热插拔，想加什么加什么 |

## 开源了

项目叫 **Voice Wake**，MIT 协议，随便用随便改。

> 🔗 GitHub：https://github.com/Czy050412/voice-wake

```
git clone https://github.com/Czy050412/voice-wake.git
cd voice-wake
install.bat    # 一键安装
start.bat      # 启动
```

新电脑上3分钟跑起来：`install.bat` 自动创建虚拟环境、装依赖、下载模型，然后双击 `start.bat` 就完事。

## 还没完：顺便做了屏幕操控 Skill

语音助手搞完了，我又顺手写了个 WorkBuddy Skill——**screen-automation**：

- 截全屏/局部截图
- 鼠标点击、双击、右键、拖拽、滚轮
- 键盘输入、快捷键、组合键
- 窗口查找、切换到前台

配合视觉模型（比如Claude、GPT-4V），AI 可以直接"看懂"你的屏幕，然后用这套 skill 操控你的鼠标键盘——**真正的AI远程手。**

不过目前我用的模型还不支持读图，等换了多模态模型，这个组合拳才算完整。
